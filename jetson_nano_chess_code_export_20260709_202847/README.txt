Jetson Nano 下棋相关代码导出

导出时间：2026-07-09 20:28:47
远端：jetson@172.20.10.11

文件说明：
- README.txt：本导出包说明、当前进度、待解决问题和文件用途。
- dofbot_web_control.py：Jetson Nano 上运行的 Web 后端，负责网页接口、机械臂抓取/放置动作、棋盘扫描、赛道 2 下棋流程入口。
- track2_game_ai.py：下棋决策 AI 的代码实现。Web 后端扫描棋盘后，会把棋盘状态传给这里，由它决定下一步从哪里取棋、放到哪里。
- dofbot_cell_poses.json：9 宫格棋盘的固定机械臂参数/姿态数据，包含各格子的抓取和放置角度配置。
- dofbot_board_view.json：棋盘视觉配置，用于辅助识别棋盘区域和扫描视角。
- 赛道二规则.md：比赛规则、算法设计和具体下棋策略说明。我们实际采用的下棋策略已经写在这个文件里；track2_game_ai.py 是对应代码实现。
- 棋盘(1)_等分.pdf：当前使用的等分 9 宫格棋盘纸文件，用于打印/校准实际棋盘位置。
- EXPORT_MANIFEST.json：导出清单，记录文件大小、远端修改时间和 sha256 校验信息，方便核对文件是否被改动。

当前进度：
1. 抓取与放置
   - 已经成功实现从一个棋盘格子抓取棋子，并放置到另一个棋盘格子。
   - 当前抓取/放置主要通过固定的机械臂电机角度参数控制，不再依赖上一次记录的 last pose 或 last pick。
   - 9 宫格的抓取和放置参数保存在 dofbot_cell_poses.json，同时核心执行逻辑在 dofbot_web_control.py 中。

2. 下棋策略
   - 赛道 2 的下棋策略已经完成设计，决策代码在 track2_game_ai.py 中。
   - Web 端不仅仅调用策略，还负责完整下棋流程：选择先手/后手、选择我方颜色（蓝色或黄色）、开始下棋、下一步下棋、重置棋局、显示棋盘状态。
   - 先手和后手会影响下棋流程。先手第一次可以从机械臂起点位置直接取棋并落子；后手或后续回合需要先扫描棋盘，再根据棋盘局势决定抓取和放置。
   - 蓝色和黄色会影响我方/敌方棋子的识别和策略输入。
   - 先手和后手对应的机械臂起点位置可能不同，后续如果比赛摆放方式变化，需要分别校准起点参数。

当前待解决问题：
- 下棋时出现过扫描不到敌方棋子的情况。例如我方先手落中间后，敌方放到左上，点击“下一步”时没有识别到左上的敌方棋子。
- 目前怀疑问题可能在扫描结果没有正确传到 Web 端，或者扫描/识别结果没有正确进入下棋状态。后续需要重点检查 scan_board_state 的返回结果、/api/track2_next 的扫描结果解析，以及前端实时棋盘显示的数据来源。
- 也需要确认扫描位姿和颜色阈值是否适配当前等分棋盘纸，避免视觉扫描实际没有对准对应格子。

说明：

- 导出过程没有执行机械臂动作。
