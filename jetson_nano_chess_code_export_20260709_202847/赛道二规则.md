# 赛道二：机械臂动态移动博弈竞赛

适用设备：DofBot机械臂、DOFBOT SE

棋子分为蓝色和黄色两种。开局前会告知队伍本局为先手还是后手。

每支队伍手里有且仅有 3个物理棋子。

- **阶段一（落子期）：** 比赛开始，双方轮流从备料区抓取棋子放入棋盘空位，直到双方各自在场上放满 3 个棋子。如果期间有人达成三连线，则直接获胜。
- **阶段二（移动期）：** 若 6 个棋子全部落完仍未分出胜负，比赛进入高潮。接下来的回合中，机械臂不能再拿新棋子。双方进入新博弈阶段，轮流移动棋盘上的每个棋子，然后再次进行比试（前先手变后手）。交替进行，直至决出胜负。

- **注意：** 一共2轮。如果2轮各方均未出现失误。如果打平，根据表现，获得良好或中等成绩；如果决出胜负，根据表现，则胜利方为优秀或良好，失败方为中等或及格。

由于机械臂需要“在棋盘内部进行二次抓取”，物理碰撞风险增高，制定硬性规则：

- **触碰他方棋子：** 机械臂在抓取或放下己方棋子时，若机械爪、关节或棋子晃动碰到了其他棋子并造成物理位移，视为违规，当场判负。
- **无效回合与原地踏步：** 算法不能选择“把格子 1 的棋子抓起来，又放回格子 1”。每次移动必须改变棋盘的实际状态，否则直接判负。
- **抓取掉落：** 如果在抓取过程中，棋子掉落，允许将棋子放回，重新抓取。**放置越界：** 如果棋子放置时有压线情况，允许将棋子放回，重新抓取。
- 每场比赛总计允许3次重新抓取放回，超过3次，直接判负。

## 算法设计

本赛道可以看作“九宫格三子棋 + 落满后的移动博弈”。棋盘按 1-9 编号：

```text
1 2 3
4 5 6
7 8 9
```

程序内部用 9 个格子的数组表示棋盘：

- `0`：空格
- `1`：蓝棋
- `2`：黄棋

每一步决策分为两个阶段：

1. **落子期：** 双方各有 3 个物理棋子，未放满 6 个棋子前，只能从备料区抓取己方颜色棋子放到空格。
2. **移动期：** 双方 6 个棋子全部在棋盘上后，不能再拿新棋子。每回合选择一个己方棋子，从原格移动到一个空格，且 `src != dst`，禁止原地放回。

每次选择动作时按以下优先级：

1. **立即获胜：** 如果当前有一步可以形成三连线，直接执行。
2. **立即防守：** 如果对方下一步能形成三连线，优先占住关键空位。
3. **极大极小搜索：** 对剩余合法动作做深度搜索，评分中心、角点、双连威胁和胜负结果，选择综合分最高的一步。
4. **移动期约束：** 只枚举己方棋子的移动，不会选择对方棋子；移动目标必须为空；源格和目标格必须不同。

胜利线共 8 条：

```python
(1, 2, 3), (4, 5, 6), (7, 8, 9),
(1, 4, 7), (2, 5, 8), (3, 6, 9),
(1, 5, 9), (3, 5, 7)
```

## 代码实现

完整算法代码已写在：

```text
track2_game_ai.py
```

这个文件只负责棋局决策，不直接控制摄像头和机械臂。视觉模块负责输出 9 格棋盘状态，机械臂模块负责把算法返回的动作转换成抓取坐标。

核心调用示例：

```python
from track2_game_ai import BLUE, YELLOW, Track2AI, parse_board

# 假设本队是蓝棋，并且蓝棋先手
ai = Track2AI(ai_color=BLUE, first_player=BLUE, max_depth=8)

# "." 表示空格，"B" 表示蓝棋，"Y" 表示黄棋
board = parse_board(".........")

# 返回 Move 对象。落子期 src=None，移动期 src/dst 都有值。
move = ai.choose_move(board, to_move=BLUE)
print(move.to_cells())
```

输出示例：

```python
{"kind": "place", "color": "blue", "src": None, "dst": 5}
```

移动期示例：

```python
from track2_game_ai import BLUE, Track2AI, parse_board

ai = Track2AI(ai_color=BLUE, first_player=BLUE, max_depth=8)

# 蓝棋在 1、5、6；黄棋在 2、3、4；轮到蓝棋移动
board = parse_board("BYYYBB...")
move = ai.choose_move(board, to_move=BLUE)
print(move.to_cells())
```

输出示例：

```python
{"kind": "move", "color": "blue", "src": 6, "dst": 9}
```

这表示把第 6 格蓝棋移动到第 9 格，形成 `1-5-9` 三连线。

## 与视觉和机械臂的接口

建议把系统拆成 3 层：

1. **视觉层：** 复用 `main.py` 里的棋盘识别思路，输出长度为 9 的棋盘数组，例如 `[1, 2, 0, 0, 1, 0, 0, 0, 2]`。
2. **决策层：** 调用 `track2_game_ai.py`，输入棋盘数组、己方颜色、当前先后手，输出 `Move`。
3. **执行层：** 根据 `Move` 执行机械臂动作：
   - `kind == "place"`：从己方备料区抓取一个棋子，放到 `dst`。
   - `kind == "move"`：从棋盘 `src` 抓取己方棋子，放到 `dst`。
   - `kind == "restore"`：检测到己方棋子被移动后，把棋子从异常位置搬回原位。

机械臂执行时需要遵守硬性规则：

- 抓取前先移动到棋子正上方，再垂直下降。
- 夹稳后先垂直抬高，再平移到目标格上方，避免横向扫到其他棋子。
- 移动棋盘内棋子时只抓己方颜色棋子，禁止越过低高度区域。
- 每次执行后重新识别棋盘，确认棋盘状态确实改变。

## 违规和异常检测

`track2_game_ai.py` 提供两个辅助函数：

```python
transition_move(previous, observed, actor_color)
detect_tamper(previous, observed, protected_color)
```

用途：

- `transition_move`：判断对方刚才的棋盘变化是否是一次合法落子或移动。
- `detect_tamper`：如果识别到己方棋子被移动，返回一个 `restore` 动作，让机械臂把己方棋子搬回原位。

示例：

```python
from track2_game_ai import BLUE, detect_tamper, parse_board

previous = parse_board("BYYYBB...")
observed = parse_board("BYYY.B..B")

correction = detect_tamper(previous, observed, BLUE)
print(correction.to_cells())
```

输出：

```python
{"kind": "restore", "color": "blue", "src": 9, "dst": 5}
```
